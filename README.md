# ДЗ 6 — Профилирование и оптимизация кода

Полный текст задания: https://advanced-python.ru/advanced-python/homework/hw06

## Быстрый старт

1. **Заполни `STUDENT.md`** — впиши ФИО точно как в ведомости (от этого зависит вариант).
2. **Установи зависимости:**
   ```bash
   uv sync
   ```
3. **Посмотри свой вариант:**
   ```bash
   uv run pytest -v --co
   ```
4. **Профилируй медленный код** в `src/slow_code.py` (задание 6.1).
5. **Реализуй оптимизации** в `hw06/optimized.py` (задание 6.1) и `hw06/accelerated.py` (задание 6.2).
6. **Заполни отчёт** в `hw06/REPORT.md`.
7. **Проверь локально:**
   ```bash
   uv run ruff check .
   uv run ruff format --check .
   uv run pytest -v
   ```
8. **Запуш в main** — CI проверит автоматически.

## Задания

### 6.1 — Профилирование и итеративная оптимизация (3 балла)

В файле `src/slow_code.py` находится намеренно медленная реализация одного из алгоритмов
(какой именно — зависит от вашего варианта).

| Вариант | Задача | Описание |
|---------|--------|----------|
| 0 | Алгоритмы на графах | Кратчайшие пути, наивный O(n^3) |
| 1 | Обработка CSV | Агрегация без pandas, наивные циклы |
| 2 | Числовые вычисления | Матричные операции с вложенными циклами |
| 3 | Парсинг текста | Поиск паттернов в большом тексте |

**Подсказка для варианта 2** — три естественные итерации:
1. Вычислять норму строки один раз (а не заново для каждого элемента)
2. Заменить умножение матриц на `numpy.dot` / оператор `@`
3. Полностью векторизованный пайплайн: генерация данных + matmul + нормализация на numpy

**Порядок работы:**

1. Профилируй исходный код с помощью `cProfile` и `pyinstrument`.
2. Оптимизируй в **минимум 3 итерации**, каждый раз профилируя заново.
3. Добейся ускорения **минимум в 4 раза** по сравнению с оригиналом.
4. Заполни отчёт `hw06/REPORT.md` — для каждой итерации укажи:
   - Результат профилирования (узкое место)
   - Что было изменено
   - Полученное ускорение

### 6.2 — Ускорение через Numba/Cython (2 балла)

Возьми **исходный** медленный код (из `src/slow_code.py`) и ускорь его критичный участок
с помощью `numba` (`@jit`) или Cython. Реализуй в `hw06/accelerated.py`.

> **Важно:** ускоряй именно оригинальный алгоритм (циклы, логику), а не переписывай
> всё на numpy. Смысл задания — показать, что компилятор JIT/AOT даёт выигрыш на
> «наивном» коде без смены алгоритма.

Сравни 3 версии:
- Оригинал (`src/slow_code.py`)
- Оптимизированная (`hw06/optimized.py`)
- Ускоренная (`hw06/accelerated.py`)

Ускоренная версия должна быть как минимум в **3 раза быстрее** оригинала.
